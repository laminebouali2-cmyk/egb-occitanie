## 1. Analyse du marché SEO

### La réalité du terrain
Le remplacement de pare-brise est un marché **hyper local** et **hyper intentionniste**. Celui qui cherche "remplacement pare-brise Toulouse" a besoin du service MAINTENANT. Le taux de conversion organique sur cette requête est parmi les plus élevés du web local.

### Le monstre en face : Carglass
Carglass domine le SEO avec :
- Domain Authority > 50
- Des milliers de pages locales ("Carglass Toulouse", "Carglass Blagnac", etc.)
- Budget SEO illimité
- Notoriété de marque (les gens tapent "Carglass" directement)

### Notre stratégie : ne pas les affronter en face
On ne va pas battre Carglass sur "pare-brise Toulouse". Mais on peut :
1. **Gagner sur les mots-clés long-tail** qu'ils ne ciblent pas
2. **Dominer le Google Local Pack** (Google Maps) — plus facile pour un indépendant local
3. **Capturer les recherches mobiles géolocalisées** ("pare-brise près de moi")
4. **Créer du contenu expert** que Carglass ne fait pas (trop corporate)

## 4. SEO technique

### Architecture des URLs

```
✅ shedliauto.com/remplacement-pare-brise
✅ shedliauto.com/intervention-mobile
✅ shedliauto.com/prise-en-charge-assurance

❌ shedliauto.com/services/pare-brise/remplacement
❌ shedliauto.com/page/2
❌ shedliauto.com/?p=123
```

**Règles :**
- URLs courtes, descriptives, en minuscules
- Tirets (-) comme séparateurs
- Pas de dossiers profonds (/a/b/c/d)
- Pas de paramètres d'URL
- Trailing slash : non (Next.js default)

### Balises obligatoires par page

```html
<!-- Chaque page doit avoir : -->
<title>Unique, < 60 caractères, mot-clé en premier</title>
<meta name="description" content="Unique, 150-160 chars, avec CTA implicite" />
<link rel="canonical" href="https://shedliauto.com/page-url" />

<!-- Open Graph -->
<meta property="og:title" content="..." />
<meta property="og:description" content="..." />
<meta property="og:image" content="https://shedliauto.com/og/page.png" />
<meta property="og:type" content="website" />
<meta property="og:url" content="https://shedliauto.com/page-url" />

<!-- Twitter -->
<meta name="twitter:card" content="summary_large_image" />
```

## 5. Google Business Profile (hors site mais critique)

### Pourquoi c'est prioritaire
Le **Local Pack** (les 3 résultats Google Maps) capte **42% des clics** sur les recherches locales. Un Google Business Profile optimisé peut générer plus de trafic que le site lui-même.

### Optimisations à faire

| Action | Priorité |
|--------|----------|
| Créer/revendiquer la fiche Google Business | Critique |
| Catégorie principale : "Service de réparation de pare-brise" | Critique |
| Photos professionnelles (10 minimum) | Haute |
| Horaires complets | Haute |
| Répondre à chaque avis Google | Haute |
| Poster des updates mensuels (photos chantier) | Moyenne |
| Ajouter les services avec description | Moyenne |
| Zone de service définie précisément | Haute |

### Collecte d'avis systématique

Après chaque intervention :
1. SMS automatique avec lien direct vers la page d'avis Google
2. Le lien doit être un **lien raccourci Google** (g.page/...)
3. Objectif : **1 avis par semaine minimum**
4. Toujours répondre aux avis (positifs et négatifs)

## 6. Performance web = SEO

Google utilise les Core Web Vitals comme facteur de classement.

### Cibles techniques

| Métrique | Cible | Levier |
|----------|-------|--------|
| LCP < 2.5s | Image hero en WebP, `priority`, CDN | next/image |
| FID < 100ms | Pas de JS bloquant, server components | Next.js App Router |
| CLS < 0.1 | Dimensions d'image explicites, fonts preloaded | next/font + width/height |
| TTFB < 600ms | Static generation, edge deployment | Vercel |

### Bonnes pratiques Next.js pour le SEO

1. **Server Components** par défaut → HTML complet au premier rendu → Google indexe tout
2. **Static Generation** (`generateStaticParams`) pour les pages qui ne changent pas
3. **next/font** pour charger Manrope → pas de FOIT/FOUT → pas de CLS
4. **next/image** pour toutes les photos → WebP, lazy, responsive, blur placeholder
5. **Metadata API** → chaque page exporte son `metadata` (titre, description, OG)
6. **Pas de client-side rendering** pour le contenu principal → tout est dans le HTML initial
